1. The SWITCH Instruction
"The Relocation"

The SWITCH instruction updates the Namespace Register (CR15). This defines the "Universe" of accessible objects for the process.

Input: A General Register (CR<reg>) holding a capability to a C-List (The new Namespace).

Validation:

Integrity: The key must not be NULL.

Permission: The key must have the Enter (E) permission. This confirms the user has the right to "step inside" this new environment.

The Action: The machine copies the capability from the General Register into CR15.

The Result: The previous Namespace is unmapped. Future LOAD operations will now resolve symbolic names or indices against this New Namespace.

2. Haskell Implementation
Add this to Section 4 (Instruction Set) of your Replit code.

Haskell

-- --- INSTRUCTION: SWITCH ---
-- 1. Checks for 'Enter' permission on the source capability.
-- 2. Updates CR15 (The Universe) to the new Namespace.
instrSWITCH :: CPUState -> Int -> Either String CPUState
instrSWITCH cpu srcIdx = 
    let srcCR = Map.findWithDefault emptyCR srcIdx (c_regs cpu)
    in 
    if cachedName srcCR == "NULL" 
        then Left "TRAP: Target Namespace is NULL"
    else if not (PermEnter `elem` activePerms srcCR) 
        then Left "TRAP: No ENTER Permission on Namespace Key"
    else 
        Right cpu { cr15_NS = srcCR }

Add this to Section 5 (Console/Help).

Update showHelp:

Haskell

    putStrLn "| SWITCH <reg>           | Set CR15 (Namespace) to Cap in reg          |"
Update runConsole:

Haskell

        -- SWITCH <reg>
        ("SWITCH":rStr:_) -> 
            case readInt rStr of
                Just r -> case instrSWITCH cpu r of
                    Right newCpu -> do
                        putStrLn $ "   [OK] Namespace Switched to: " ++ cachedName (cr15_NS newCpu)
                        runConsole newCpu
                    Left e -> putStrLn ("[TRAP] " ++ e) >> runConsole cpu
                Nothing -> putStrLn "[ERROR] Invalid Register Index" >> runConsole cpu
